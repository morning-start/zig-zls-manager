## 🎯 项目愿景

构建一个专业级的 **Zig + ZLS 联合版本管理 CLI 工具**，成为 Zig 生态系统中最优秀的版本管理解决方案。

### 核心价值主张

1. **ZLS 作为一等公民**: 独立的 ZLS 管理能力，而非附属功能
2. **智能兼容性**: 自动检测和警告版本不匹配问题
3. **开发者体验**: 一键初始化、IDE 集成、项目级配置
4. **跨平台一致性**: Windows/macOS/Linux 统一的用户体验
5. **高性能与安全**: Rust 实现 + SHA256 校验 + 安全下载

## ✅ Phase 2: 增强版 (v0.2.0) - 已完成

**目标**: 完善独立 ZLS 管理能力，增加项目级配置和多 IDE 支持

**状态**: ✅ 已完成

### 2.1 新增功能 - 全部完成

#### A. 独立 ZLS 版本管理
- ✅ `zzm zls install` / `uninstall` / `use` / `list` - 完整独立管理

- ✅ 内置兼容性规则数据库

## ✅ Phase 2.5: 稳定版 (v1.2.1) - 已完成

**目标**: 优化性能、重构架构、完善所有功能

**状态**: ✅ 已完成

### 2.5.1 核心架构优化 - 全部完成

#### A. 泛型彻底化
- ✅ `ToolIndexEntry` 统一数据结构
- ✅ 消除 15+ `match` 分支
- ✅ `InstalledIndex` 统一管理 Zig/ZLS 版本

#### B. Core 层输出解耦
- ✅ `InstallCallbacks` 回调方案
- ✅ 支持 `--json` 纯净输出
- ✅ 为 TUI/GUI 做准备

#### C. Commands 层数据抽象
- ✅ `OutputDispatcher` 统一输出调度
- ✅ 消除重复的 map + render 逻辑

#### D. 其他优化
- ✅ AppContext OnceCell 懒加载
- ✅ PostInstallHook Trait 抽象
- ✅ 索引读取合并优化

### 2.5.2 性能与稳定性提升 - 全部完成

#### A. 性能优化
- ✅ 冷启动 < 100ms（优化前 > 200ms）
- ✅ 并行下载（Zig + ZLS 使用 `tokio::join!`）
- ✅ 流式 SHA256 校验（内存恒定）

#### B. 稳定性增强
- ✅ 原子性安装与回滚（ZLS 失败时回滚 Zig）
- ✅ 边界情况处理（网络中断、磁盘空间不足）
- ✅ 健壮性测试（231 个测试全部通过）

#### C. 代码质量
- ✅ 代码重复度降低 50%+
- ✅ `cargo clippy -D warnings` 零警告
- ✅ 231 个测试全部通过（214 单元 + 16 集成 + 1 文档）

## 📝 决策记录

### ADR-004: Phase 优先级决策
**决定**: 渐进式演进，先做核心功能再扩展

### ADR-005: IDE 支持优先级
**决定**: VS Code → Neovim/Helix → JetBrains

### ADR-006: 泛型彻底化
**决定**: 引入 `ToolIndexEntry` 统一数据结构，消除所有 `match` 分支

### ADR-007: 输出解耦方案
**决定**: 使用回调方案而非事件流，简单且满足 CLI 场景需求
